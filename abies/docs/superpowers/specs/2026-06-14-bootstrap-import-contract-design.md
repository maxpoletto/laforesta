# Bootstrap & import contract — design

- **Date:** 2026-06-14
- **Status:** Draft for review
- **Scope:** sub-project 1 of 2 (the abies-side import contract). The external
  La-Foresta data producer is sub-project 2 (separate spec).

## 1. Problem & goals

Today abies is coupled to pdg-2026 / La Foresta in its data-loading path:

- Two divergent load paths: `make dev` runs 9 bespoke management commands (8
  `import_*` + `build_geo`) against a hand-curated `../abies-data`;
  `bin/bootstrap-data` runs only 4 against the messy `../bosco/data`. Dev and
  prod load different data, in different amounts, from differently-named files.
- The importers hard-code La-Foresta specifics: `REGIONS`, `ECLASSES`,
  `TRACTORS`, `PRODUCT_MAP`, `NOTE_FLAG_MAP`, three divergent species maps
  (`GENERE_MAP` ×2, `SPECIES_COL_MAP`), one-off names/dates/years
  (`SURVEY_NAME`, `GRID_NAME`, `PLAN_NAME`, `2026/2040`, `r_m=20`,
  `DEFAULT_SAMPLE_DATE`), synthetic `'X'` parcels, and a parallel `*_LEGACY`
  column vocabulary.
- This bespoke path duplicates logic that **already exists** in the in-app CSV
  importers (`grid_csv_import_view`, `tree_csv_import_view`,
  `plan_csv_import_view`, `mark_csv_import_view`, `hypso_params_import`), which
  define strict, canonical, user-facing CSV formats with row/column error
  reporting.

**Goal:** decouple abies (a generic, reusable forestry app) from La Foresta /
pdg-2026 (one specific customer and planning cycle). Define a single, strict,
documented, canonical CSV+GeoJSON import contract; load it via one configurable,
guided `bootstrap` command that rides on the in-app importers; and make an abies
instance **portable** via a lossless CSV bundle.

Non-goals (this spec): generating canonical inputs from the existing messy La
Foresta data (pdg-2026 outputs + Excel exports) — that is sub-project 2. Building
the in-app harvest *import view* — only its shared core is built here (the view
is an upcoming feature that will reuse the core).

## 2. North star: lossless round-trip

> The data dir **is** a full-instance CSV+GeoJSON snapshot. `export <dir>` from
> instance A produces exactly what `bootstrap <dir>` consumes to stand up
> instance B, losslessly. Import format ≡ export format ≡ canonical, for every
> domain.

"Lossless" is modulo:
- **Derived values** (volume, mass) — abies recomputes them from primitives, so
  the snapshot need only carry primitives. Derived columns may appear in a CSV
  (because the table displays them) but are **ignored on import and recomputed**;
  re-export reproduces the same numbers, so round-trip is still identity.
- **Geometry** — the bundle carries **source** geometry only. The enriched
  served GeoJSON (`data/geo/`, with the derived `coppice` property) is a
  generated artifact, regenerated by `build_geo` at bootstrap, and excluded from
  round-trip identity. Geometry is never stored in the DB, so on export "source"
  = the served file with derived properties stripped.
- **Audit history / timestamps / versions** — per-instance, not part of the
  snapshot.

This requirement is also the strongest correctness test we have: a
`export → bootstrap → export → diff` round-trip must be the identity (§9).

## 3. Design principles (decided)

1. **One strict canonical format per domain.** No aliasing, no legacy headers,
   no silent defaults for semantically significant fields. Missing required
   column/value → hard error naming row+column. The burden of producing
   conformant inputs is **upstream** (the producer / the exporting instance).
   - Concretely: `Raggio` (sample-area radius) becomes **required** — the old
     `r_m=20` default silently applied a wrong radius until hard-coded; we drop
     the default and require the column.
2. **Importers are strict loaders, not normalizers.** All value normalization
   (species synonyms like `Pino`→`Pino Laricio`, flag-string decoding) lives in
   the producer. abies requires canonical values and errors on unknown.
3. **Hybrid reference sourcing.** Instance-specific reference (regions, eclasses,
   tractors, crews) comes from the data dir. Generic reference (species,
   products) ships as in-repo defaults **and** is an *optional* data-dir input
   (an export bundle carries the instance's edited species/products; absent →
   in-repo default seed).
4. **Empty-DB-first, guarded.** `bootstrap` loads cleanly into an empty
   instance; against a non-empty instance it refuses by default (naming what
   exists) and requires `--force` to wipe-and-reload. The app is the source of
   truth after bootstrap; the data dir is the source of truth only at bootstrap.
5. **Convergence.** Each domain's CSV import/export core is shared by the view
   (container chosen in the UI) and `bootstrap` (container named in the data).
   The 8 pdg `import_*` commands are deleted; `build_geo` is kept (already
   clean) and reused by `bootstrap`/`export`.
6. **abies computes derived quantities** (volume via Tabacchi, mass via density)
   from measured primitives. Derived columns are not authoritative input.
7. **One format per table, scoped vs unscoped.** The bundle is every table's
   export, unfiltered. Each bulk table's canonical CSV always carries its
   container-key column (`Survey`/`Grid`/`Plan` name); the on-screen export and
   the bundle therefore share one schema. The shared core resolves the container
   from an explicit parameter (UI) or from the per-row column (bundle).

## 4. Architecture

```
                    apps/<domain>/csv_import.py  (shared core, per domain)
              parse → validate(staged index) → apply(rows, container?, mode)
                              ▲                      ▲
        view (container from form/URL)     bootstrap/export (container from data)
```

- **Shared cores.** Extract each domain's CSV core out of its view into a shared
  module (`apps/<domain>/csv_import.py`), parameterized by target container +
  mode, with an explicit three-phase contract:
  - **parse** — CSV bytes → rows (dialect auto-detected).
  - **validate** — no DB writes of its own: checks required cols/types and
    resolves FKs against an injected `*Indexes` (built from the live DB).
    Returns row/column errors plus parsed rows with FK targets resolved to
    objects, ready for `apply`.
  - **apply** — writes, assuming validated input.

  **Load model (decided 2026-06-14, atomic-transaction).** Because `validate`
  resolves FK targets to *saved* objects, the orchestrator loads files in
  dependency order inside one `transaction.atomic()`: it validates each file and
  applies its valid rows immediately, so a later file's `validate` resolves
  against the rows an earlier file just wrote. Errors are accumulated across all
  files (not fail-fast); on **any** error — or for `--check` — the whole
  transaction rolls back, so there is never a half-loaded instance and `--check`
  is observationally a no-write dry-run. (An earlier draft of this spec
  specified pure staged-index validation with no rollback and called the
  transaction approach a "hack"; the cores settled on resolving FK objects
  during `validate`, which makes the single-transaction load the simpler,
  equivalent choice.) Domains: parcels, sample grid, surveys/sampled-trees,
  preserved-trees (PAI), harvest plans/items, harvests (new), hypso (already
  extracted as `hypsometry.parse_param_csv` / `replace_active_set`).
- **`manage.py bootstrap <data_dir> [--check] [--force]`.** Thin orchestrator,
  all inside one `transaction.atomic()` (rolled back on any error or `--check`):
  1. Seed reference (regions, eclasses, tractors, crews; species/products from
     data dir or in-repo default), then named containers (surveys, sample grids,
     harvest plans), then each bulk file via its domain core — validating and
     applying each file's valid rows in dependency order, accumulating errors.
  3. Emit a structured report (§7).
  The per-domain cores remain individually callable for targeted dev re-imports.
- **`manage.py export <dir>`.** The dual: writes the canonical bundle (every
  table's CSV in canonical format + `terreni.geojson`). Enables the round-trip
  test and instance portability. (An in-app "Export all" can wrap this later.)
- **Deletions.** The 8 pdg `import_*` management commands (`import_reference`,
  `import_parcels`, `import_mannesi`, `import_sample_grid`,
  `import_sampled_trees`, `import_pai`, `import_calendar`, `import_hypso_params`
  — note `import_reference`/`import_parcels` fold into reference/parcel seeding);
  `GENERE_MAP` ×2, `SPECIES_COL_MAP`, `TRACTOR_COL_MAP`, `PRODUCT_MAP`,
  `NOTE_FLAG_MAP` and all flag-string decoding; `_AliasedCsv`,
  `GRID_CSV_ALIASES`, `_HIGHFOREST/_COPPICE` alias maps; the `*_LEGACY` column
  constants; synthetic-`'X'`-parcel creation; hard-coded `REGIONS`/`ECLASSES`/
  `TRACTORS`/names/dates/years/`r_m`/`shoot`/`preserved` defaults.

## 5. Data dir contract (file-by-file)

All files: UTF-8, ISO dates, localized headers (`_it`/`_en` constants — never
inline strings). The **bundle** is written in one canonical CSV dialect
(`,`-field / `.`-decimal, the project's machine-canonical convention); round-trip
identity is defined over that dialect. (Import still auto-detects either dialect
via `csv_io`, and the on-screen per-table export stays locale-driven — same
schema, only the delimiter/decimal differ.) Column literals below are finalized
against the existing in-app **export** output during implementation (export
defines the template). This section lists only the **canonical contract**; which
legacy/unread columns each *current* file happens to carry is an implementation
detail tracked in the plan, not here.

### Required (minimum viable instance)

| `eclasses.csv` | Eclass | `Comparto`, `Ceduo` (bool) | replaces hard-coded `ECLASSES`; `min_harvest_volume` if used |

## 6. New canonical harvest format (decision 1b) & region-wide rows (2a)

**`harvests.csv` — single file, one row per harvest; species/tractor splits as
dynamic, data-driven columns** (not a compound mini-format inside a cell).

Static columns: `Compresa`, `Particella` (blank ⇒ region-wide), `Data`,
`Squadra`, `Tipo` (product, canonical), `Q.li` (mass, **primitive**), `VDP`,
`Prot.`, explicit `Danneggiato` / `Fitosanitario` / `PSR` booleans, `Altre note`.

Dynamic columns (localized prefixes `Specie:` / `Trattore:`):

- `Specie: <Genere>` — percentage for that species; `<Genere>` must exist in
  `species.csv`.
- `Trattore: <Trattore>` — percentage for that tractor; `<Trattore>` must exist
  in `tractors.csv` (its unique `Trattore` key column — manufacturer+model is not
  unique once year matters).

Rules:
- Blank ⇒ 0; non-blank ⇒ numeric percentage in range.
- Species percentages must sum to 100 when `Q.li > 0`.
- Tractor percentages either sum to 100 when any tractor is present, or are all
  blank/0 if unknown.
- Duplicate normalized headers (same species/tractor twice) are a hard error.
- Unknown `Specie:`/`Trattore:` key is a hard error (strict; no normalization).
- Export writes these columns deterministically in `species.csv` / `tractors.csv`
  order, so the round-trip is stable.

abies **computes** `volume_m3` from `Q.li` + species densities; no volume column
is authoritative on import.

**Region-wide rows (catastrofato), decision 2a:** in the data, a blank
`Particella` means "applies to the whole region" → stored as `parcel=NULL`,
`region=<Compresa>`. `HarvestPlanItem.parcel_id` is already nullable with a
`region_id` for this. `Harvest` is **not**: this spec adds a migration making
`Harvest.parcel` nullable and adding `Harvest.region` FK (mirroring
`HarvestPlanItem`), with a validity constraint "exactly one of parcel/region".
Convention: such rows render as `Particella = "X"` in user-visible displays
(plans, etc.), but `"X"` is never a stored parcel.

## 7. The `bootstrap` command

- **Args:** `<data_dir>` (configurable; replaces the `DATA_DIR ?= ../abies-data`
  default contract). `--check` = validate + report only, no writes. `--force` =
  wipe-and-reload (required when the instance is non-empty).
- **Atomic load with error accumulation (§4 load model):** inside one
  `transaction.atomic()`, process every present file in dependency order —
  validate it via its domain core, record any row/column errors, and apply its
  valid rows so later files resolve FKs against them. Do **not** fail fast:
  keep going so the report lists errors from every file at once. Required-file
  missing → hard error. Optional-file missing → skip + note. Malformed/unresolved
  row → error naming file, row, column, raw value (reusing `csv_io` + `ERR_CSV_*`).
- **Rollback discipline:** if the run accumulated **any** error, or it is a
  `--check` run, roll the transaction back — never a half-loaded instance, and
  `--check` persists nothing. The transaction commits only on a clean,
  non-`--check` run.
- **Guard:** non-empty instance without `--force` → refuse, list populated
  domains, exit non-zero.
- **Report (stdout):** per file → present/missing, required/optional, rows
  loaded, errors. Final summary line with totals. `--check` prints the same
  report with "would load" counts.
- **Ordering:** reference → named containers → bulk (FK dependencies respected).
- **Wiring:** `make dev` calls `bootstrap <dir> --force`; `make import` /
  `import-base` collapse into it; `bin/bootstrap-data` calls `bootstrap` against
  one canonical data dir on the VM (replacing the 4-command subset).

## 8. The `export` command (decision 4a)

- `manage.py export <dir>` writes the canonical bundle in the canonical CSV
  dialect (§5): every table's CSV in the format §5 defines, plus **source**
  `terreni.geojson` (the served enriched GeoJSON is derived and not in the
  bundle, §2). Same per-table schema as the on-screen export (§3.7) — derived
  columns may be present but are non-authoritative; `bootstrap` ignores and
  recomputes them, so the round-trip is identity. One format, not two.
- Reuses each domain's shared export core; the same cores back the existing
  per-table on-screen exports (which gain the always-present container column,
  §3.7) and a future in-app "Export all".

## 9. Testing

- **Round-trip identity (headline test):** seed a known instance → `export d1`
  → wipe → `bootstrap d1` → `export d2` → assert `d1 == d2` (modulo audit/
  timestamps). This is the contract's correctness guarantee.
- **Per-domain strict-validation tests:** missing required column/value, unknown
  species/region/parcel, bad/missing radius, malformed dates, region-wide rows,
  dynamic `Specie:`/`Trattore:` columns (sum-to-100, range, duplicate header,
  unknown key), `--check` dry-run, `--force` guard, non-empty refusal.
- **Fixture:** a small, hand-built **canonical** data dir under `test/data/`.
- **Transition fixture:** a one-time conversion of the existing curated
  `../abies-data` (legacy/pdg format) into canonical, so `make dev` keeps
  working before sub-project 2 exists. (The general producer is sub-project 2;
  this is just enough to bootstrap the real instance.)
- JS/Python existing suites (`make test`) stay green.

## 10. User-facing doc

`docs/bootstrap.md`: the canonical contract for operators/producers — the
file-by-file column tables (§5), the harvest split-cell + region-wide
conventions (§6), how to run `bootstrap`/`export`/`--check`, required-vs-optional,
and the round-trip guarantee. Linked from `abies/CLAUDE.md` Docs index.

## 11. Out of scope / follow-ons

- **Sub-project 2:** the external producer that converts La Foresta data
  (pdg-2026 outputs + Excel CSVs) into a canonical data dir. Owns all
  normalization (species synonyms, flag decoding, layout fixups).
- **In-app harvest import view:** upcoming feature; reuses the harvest shared

## 12. Risks & notes

- Removing aliasing makes **in-app uploads stricter** (legacy pdg headers
  rejected). Acceptable: each domain's export defines the template that import
  requires. (Accepted, decision 3ii.)
- Changing plan/harvest export to **explicit boolean flag columns** (not a `Note`
  flag string) changes those export formats. (Accepted, decision 3i.)
- Re-casing hypso headers changes the hypso import/export format. (Accepted,
  decision 3iv.)
- The `Harvest` model change is a migration touching prelievi; the
  exactly-one-of(parcel, region) constraint must be enforced in DB + cleaned
  client/server validation.
- Unifying plan fustaia+ceduo into one items file changes the plan
  import/export from a two-file zip to one file; verify the page-piano-di-taglio
  flow and digests.
- **Region-wide dedup (already correct in code):** despite the importer's
  docstring saying "dedup on `(plan, parcel, year)`", the implementation already
  branches — parcel rows upsert on `(plan, parcel, year)` and region-wide rows
  (`parcel IS NULL`) on `(plan, region, parcel, year)`. No fix needed; the
  extraction must preserve both branches. (The earlier "latent bug" note was
  based on the docstring, not the code.)

## 13. Implementation sequencing (informs writing-plans)

This spec is large; it should be implemented as several reviewable plans, not one
diff. Suggested order (each shippable, `make test` green):

1. **Shared core foundation + container-column** — refactor each existing in-app
   importer to the three-phase core (§4), accepting the container from an
   explicit param OR a per-row column; add the always-present container column to
   its export. **Keep aliasing for now** (the real `../abies-data` is still
   legacy until step 5). No bootstrap yet. (Tightest blast radius; unblocks
   everything.)
2. **`bootstrap` orchestrator + reference/container seeding** — regions,
   eclasses, tractors, crews, species/products, surveys, grids, plans; required
   vs optional; `--check`/`--force`; structured report; transaction. Wire
   `make dev`/`bin/bootstrap-data`. Delete the corresponding `import_*` commands
   as each domain is covered.
3. **`Harvest` model change + new harvest core** — nullable parcel + region FK +
   constraint + migration; the §6 harvest CSV core; bootstrap loads harvests;
   delete `import_mannesi`.
4. **`export <dir>` dual + round-trip test** — the headline correctness gate.
5. **Cleanup pass** — once the canonical fixture + bootstrap path work: convert
   the real `../abies-data` to canonical (transition fixture) and, in the same
   change, **drop aliasing** (`_AliasedCsv` / `GRID_CSV_ALIASES` /
   `_HIGHFOREST`/`_COPPICE` maps) so `make dev` never breaks; remove
   `GENERE_MAP`/`PRODUCT_MAP`/`NOTE_FLAG_MAP`/`*_LEGACY`/synthetic-`'X'` once no
   caller remains; re-case hypso; write `docs/bootstrap.md`.
