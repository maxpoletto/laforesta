{% comment %} Manual tree+sample entry form (M3d-write). Two paths share the same form, gated by the "Ceduo" checkbox: - Unchecked (fustaia, the common case): one TreeSample row. D/h/L10 live on a single row; V/m are computed live by JS. - Checked (ceduo): N TreeSamples sharing one Tree, one per pollone. D/h/L10 (per pollone) live in an indented coppice block, plus a "matricina" toggle and an "Aggiungi pollone" button. V/m are blank for coppice rows. Disabled inputs aren't submitted, so the toggle disables the inactive side rather than removing it — switching modes preserves any in-progress entry on the inactive side. The `fustaia` POST field uses the hidden+checkbox idiom inverted: the hidden carries the default ("true" = fustaia) and the checkbox overrides to "false" when checked. Server-side parsing is unchanged. "Numero albero" is a pulldown driving cross-sample tree identity: picking an existing tree reuses its tree.id (and the client locks specie / ceduo / lat / lng to that tree's values). See docs/pages/campionamenti.md §"Cross-sample tree identity". {% endcomment %}
