{% comment %} Manual tree+sample entry form (M3d-write). Two top-level branches: - New tree (no ts): "Nuovo albero campione" — every field editable, species defaults to S.SPECIES_DEFAULT_FUSTAIA / _CEDUO by parcel eclass.coppice, ceduo checkbox follows fustaia_default. - Edit tree (ts): "Modifica albero campione" — N. albero, Specie, Ceduo are read-only (rendered as static text + hidden inputs to preserve the wire format); Data, D/h/L10, lat/lng, PAI remain editable. "Salva e continua" is suppressed (batch-entry only). Sample date lives on the parent Sample row, one per (survey, area). The Data input here mirrors that value and lets the user change it; on save, tree_save_view bumps Sample.date if it differs. This replaces the legacy inline date selector above the alberi table. Fustaia / ceduo gating: the "Ceduo" checkbox toggles which subblock (.tree-fustaia-fields vs .tree-coppice-block) is visible and which inputs submit. Disabled inputs aren't submitted, so the toggle disables the inactive side rather than removing it. On edit, the checkbox is rendered as a hidden mirror so wireCeduoToggle still applies the right visibility — see {% if ts %} branch below. The `fustaia` POST field uses the hidden+checkbox idiom: a hidden "true" carries the default; a checked checkbox overrides to "false". Server-side parsing is unchanged. "Numero albero" pulldown drives cross-sample tree identity on the New path (picking a prior tree reuses its id and locks lat/lng to that tree's values). See docs/pages/campionamenti.md. {% endcomment %}
